# [Implementation Plan] 웹 UI 지원 (Vite + Vue 3) 및 Docker 배포 구성

사용자가 마크다운 텍스트를 편리하게 직접 입력하여 검증할 수 있도록 로컬 웹 UI를 개발하되, 기술 스택을 **Vite + Vue 3 (TypeScript)** 기반으로 구축하고, 전체 애플리케이션을 **Node.js 기반 Docker 컨테이너**로 실행할 수 있도록 설계합니다. 또한, 기본 **라이트(화이트) 모드**에서 **다크 모드**로 스위칭할 수 있는 기능을 제공합니다.

---

## User Review Required

> [!IMPORTANT]
> **기술 스택 변경**
> - **프론트엔드**: Vite + Vue 3 (TypeScript)를 활용해 단독 SPA(`frontend/`)를 구성합니다.
> - **의존성**: 백엔드에는 `express` 패키지가 추가되며, 프론트엔드에는 `vue`, `vite`, `typescript` 등이 사용됩니다.
> - **테마 스위칭**: CSS 변수(Variables)를 기반으로 동작하는 화이트/다크 모드 토글 컴포넌트를 탑재합니다.
> - **Docker 배포**: 호스트의 `.env`를 컨테이너 내부에 환경변수로 주입하여 실행하도록 Dockerfile 및 docker-compose 설정을 작성합니다.

### 3. 프론트엔드 (Vite + Vue 3) 개발

#### [NEW] [frontend/](file:///Users/studio-server/srv/overfit-checker/frontend) 디렉토리 생성
- `pnpm create vite frontend --template vue-ts` 명령어를 이용해 Vite + Vue 3 + TS 보일러플레이트 생성.
- **주요 UI 컴포넌트**:
  - `App.vue`: 최상위 컴포넌트. 화이트/다크 테마 토글 및 레이아웃 관리.
  - `components/ThemeSwitcher.vue`: 라이트 <-> 다크 모드 토글 버튼.
  - `components/MarkdownInput.vue`: 사용자가 마크다운 텍스트를 붙여넣고 즉시 실행할 수 있는 에디터 영역.
  - `components/AnalysisResult.vue`: 로딩 스켈레톤 애니메이션 및 과도한 설계 요소, 대안 리스트 등을 카드/시각적 바 형태로 표현하는 모던 UI 컴포넌트.
- **스타일 (`frontend/src/style.css`)**:
  - CSS Custom Variables를 기반으로 기본 화이트 테마 및 다크 테마 컬러 팔레트 구축.
  - 프리미엄 Glassmorphism 스타일 및 스켈레톤 페이드 애니메이션 적용.

---

### 4. Docker 배포 및 실행 환경

#### [NEW] [Dockerfile](file:///Users/studio-server/srv/overfit-checker/Dockerfile)
- Multi-stage build를 통해 최종 컨테이너 용량을 최소화합니다.
  1. **Builder Stage**: `frontend/` 빌드 (`pnpm build` -> `frontend/dist` 생성) 및 백엔드 빌드 (`pnpm build` -> `dist/` 생성).
  2. **Runner Stage**: 경량 Node.js 환경에서 프로덕션 의존성만 설치 후 `node dist/index.js ui`로 서버 구동.

#### [NEW] [docker-compose.yml](file:///Users/studio-server/srv/overfit-checker/docker-compose.yml)
- 환경 변수 파일 `.env`를 자동으로 연동하여 컨테이너를 실행하도록 포트(3000:3000) 및 볼륨 설정.

## Verification Plan

### Automated Tests
- `pnpm check` (lint, typecheck, test) 동작 확인.

### Manual Verification
1. `docker compose up --build` 명령어를 통해 Docker 환경에서 빌드 및 실행이 정상적으로 이루어지는지 검증.
2. `http://localhost:3000`에 접속하여 기본 화이트 테마 확인.
3. 우측 상단 테마 토글 버튼을 눌렀을 때, 깜빡임 없이 매끄럽게 다크 모드로 전환되는지 CSS 변수 확인.
4. 마크다운 텍스트를 에디터창에 입력하고 판독 요청 후, 결과 카드가 예쁘게 출력되는지 최종 확인.
